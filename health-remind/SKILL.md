# 吃药提醒
用于根据用户输入生成吃药提醒任务命令。当用户明确表达“需要提醒自己吃药 / 服药 / 用药”时，应调用本技能。

## 核心执行规则（必须严格遵循）

### 1. 需求分析
- 判断是否调用本技能时，优先基于用户最新输入识别当前轮是否存在“提醒吃药 / 服药 / 用药”的明确诉求。
- 不因历史对话中曾出现相关内容而自动触发本技能。
- 若已确定进入本技能，参数提取阶段可参考上下文补全当前输入中省略但已明确的信息。
- 用户需求都要处理，不要遗漏。

### 2 参数来源与优先级
在已确定调用本技能后，参数提取可参考当前用户输入、上下文和记忆信息进行补全。

参数信息来源优先级如下：
1. 当前用户输入
2. 上下文
3. 记忆
4. 默认规则
5. 空字符串

规则说明：
- 当前用户输入与上下文、记忆冲突时，以当前用户输入为准。
- 上下文仅用于补全当前输入中省略但在当前对话链路中已明确的信息，不得引入新的无依据时间。
- 记忆仅用于补充长期稳定偏好，不得覆盖当前用户输入中的明确信息。
- 只有在当前输入、上下文、记忆都无法提供参数时，才使用默认规则。
- 若默认规则也不适用，则 `execTime` 输出空字符串 `""`。

### 3. 能力边界
本技能只负责：
- 识别“吃药提醒”意图
- 提取提醒时间
- 将可映射的周期性时间转换为 cron 表达式
- 生成吃药提醒命令

### 4. Cron 表达式规则
`execTime` 使用标准 5 段 cron 表达式，格式如下：

分 时 日 月 周

#### 示例：
- `0 8 * * *` 表示每天 08:00 执行
- `30 20 * * *` 表示每天 20:30 执行
- `0 9 * * 1-5` 表示每周一到周五 09:00 执行
- 若当前日期为 `2026-04-20`，“明天8点提醒我吃药” -> `0 8 21 4 *`
- “每周二提醒我吃药” -> `0 12 * * 2`
- “每周提醒我吃药” -> `0 12 * * 1`

#### 生成规则如下：
- 当用户输入中包含相对日期，且输入中提供“当前日期”时，先将相对日期解析为具体日期，再生成 cron 表达式。
- 当用户输入中包含明确日期，或可解析为明确日期，且同时包含明确时间时，在当前系统仅支持 5 段 cron 表达式的前提下，按该日期时间生成按年重复执行的 cron 表达式。
- 当用户表达明确的周期性提醒需求，但未指定具体小时和分钟时，默认补全为中午 12 点。
- 当用户表达明确的周期性提醒需求，但未指定该周期对应的具体锚点时，按最小默认锚点补全：每周默认周一，每月默认 1 号，每年默认 1 月 1 日。
- 不允许臆造时间。
- 不允许输出非 5 段 cron 表达式。
- 不允许输出解释性文本到命令中。

#### 吃药提醒的模糊时间映射规则如下：
- “早上提醒我吃药” -> 默认按上午 8 点处理 -> `0 8 * * *`
- “中午提醒我吃药” -> 默认按中午 12 点处理 -> `0 12 * * *`
- “下午提醒我吃药” -> 默认按下午 2 点处理 -> `0 14 * * *`
- “晚上提醒我吃药” -> 默认按晚上 18 点处理 -> `0 18 * * *`
- “每周二提醒我吃药” -> 默认按中午 12 点处理 -> `0 12 * * 2`
- “每周提醒我吃药” -> 默认按中午 12 点处理，并按默认锚点补全为周一 -> `0 12 * * 1`
- “每月提醒我吃药” -> 默认按中午 12 点处理，并按默认锚点补全为 1 号 -> `0 12 1 * *`
- “每年提醒我吃药” -> 默认按中午 12 点处理，并按默认锚点补全为 1 月 1 日 -> `0 12 1 1 *`
- 用户未指定任何时间信息，且也未提供明确周期信息时，不做默认补全，`execTime` 输出空字符串 `""`


## 输入参数

| 字段名           | 类型             | 必填 | 默认值 | 描述 |
|------------------|------------------|----|--------|------|
| execTime          | string           | no | —      | cron表达式字符串：获取描述中的任务执行时间信息生成cron表达式；如果没有时间信息则该字段为空|


## 输出命令
示例：
```bash
cd <skill_root> && python scripts/python/health_remind.py --execTime "0 15 * * *"
```
```bash
cd <skill_root> && python scripts/python/health_remind.py --execTime ""
```

>  说明：
> - `cd <skill_root> && python scripts/python/health_remind.py` 为前面固定输出内容，后面携带参数；
> - `<skill_root>` 为技能根目录占位符，运行时替换；
> - 若无法提取出明确且可映射的周期性 cron 时间，则 --execTime 传空字符串。
> - 仅输出 bash 命令即可



## 注意事项
- 只要用户核心诉求是“提醒吃药 / 服药 / 用药”，即可使用本技能。
- 时间信息优先从用户当前输入中提取，不要依赖猜测。
- 对“早上 / 中午 / 下午 / 晚上”这类模糊时间，按本技能预设时间映射处理。
- 对明确周期性但缺少时分的提醒请求，按默认规则补全时间。
- 对完全未给出时间的提醒请求，不要默认补具体时间，必须输出空 execTime。
